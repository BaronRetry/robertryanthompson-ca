Kinoite Notes

I want to write down every step needed to set up this website so far, so that I don't forget again.

First, we install Fedora Kiniote. I used this special version of Fedora Media Writer from the download page here .

I was having some issues with the download, but after trying it a few times it worked. I also got bitten by an irritating thing Windows does when creating bootable media; Autoplay adds some weird flipped bit that causes the checksum on the bootable drive to fail. I had to open up ms-settings:autoplay and then turn off the option that says "Use AutoPlay for all media and devices".

After that the install went off without a hitch. I chose the automatic partitioning option, banishing Windows utterly from the machine (although I might bring it back in a VM later just for Excel).

As of this writing the updates I've received have brought me to Kinoite version 42.20251026.0. To see the list of all ostree commits, do

sudo ostree pull --commit-metadata-only --depth=10 fedora fedora/42/x86_64/kinoite

... and then

sudo ostree pull --commit-metadata-only --depth=10 fedora fedora/42/x86_64/kinoite

So far I've done all the system updates through the Discover front end. There was a UEFI update as well. Suspend/sleep seems to be broken on the system because if it sleeps the motherboard never wakes up and then I get one of those POST failure messages upon restart. However, sleeping and dimming the monitors works fine at least, and who really cares because it's a desktop PC.

OK, now we need to set up a Git repo on Github. I also wanted to generate SSH and GPG keys so that I can do signed commits (despite some lingering anxiety about using my identity on the internet).

First step here is to set the global Git username and password to be the same as what Github has:

git config --global user.name "Robert Ryan Thompson" git config --global user.email "baronretry@proton.me"

Then, we generate the SSH key:

ssh-keygen -t ed25519 -C "baronretry@proton.me"

... start the SSH agent ...

eval "$(ssh-agent -s)"

... add the private key ...

ssh-add ~/.ssh/id_ed25519

... and then finally emit the public key so that we can give that to Github:

cat ~/.ssh/id_ed25519.pub

The steps to set up a GPG key are similar:

gpg --full-generate-key gpg --list-secret-keys --keyid-format=long gpg --armor --export gpg --list-secret-keys --keyid-format=long gpg --armor --export KEY_GOES_HERE

OK! Next, we need to initialize a local Git repo and upload it to Github:

git status git add README.md git commit -S -m "Initial commit" git branch -M master git remote add origin git@github.com:BaronRetry/robertryanthompson-ca.git git push -u origin master

At this point, it's probably worth it to pick a text editor to start working with files in this project. I was considering Kate but I am attached to emacs. Because I anticipate using it everywhere, I install it as a layered package:

sudo rpm-ostree install emacs systemctl reboot
